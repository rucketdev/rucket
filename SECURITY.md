# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please report it responsibly.

### Preferred Method: GitHub Security Advisories

1. Go to the [Security tab](https://github.com/rucketdev/rucket/security/advisories)
2. Click "Report a vulnerability"
3. Provide details about the vulnerability

### Alternative: Email

Send details to: **security@rucket.dev**

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

### Response Timeline

- **Acknowledgment**: Within 48-72 hours
- **Initial Assessment**: Within 1 week
- **Resolution Target**: Depends on severity

### Disclosure Policy

- We will coordinate disclosure timing with you
- Credit will be given to reporters (unless anonymity is requested)
- Please do not publicly disclose until we've had time to address the issue

## Security Best Practices for Users

- Always use the latest version
- Use strong, unique credentials
- Run behind a reverse proxy with TLS in production
- Restrict network access to trusted clients
